2. A single `<script>` block containing these functions (all pure, no DOM access):

**normalizeText(raw)** — returns cleaned string:
- Replace `\u00A0` (non-breaking space) with regular space
- Remove `\u00AD` (soft hyphen)
- Replace `\u2013` and `\u2014` (en-dash, em-dash) with ASCII hyphen `-`
- Remove `\u200B` (zero-width space)
- Trim whitespace

**validateIBAN(iban)** — returns boolean (MOD-97 per ISO 13616):
- Input: normalized IBAN string (no spaces, uppercase)
- Rearrange: move first 4 chars to end
- Convert letters to digits: A=10, B=11, ..., Z=35
- Iterative mod-97 (avoid BigInt): loop through digit string, `remainder = (remainder * 10 + parseInt(ch, 10)) % 97`
- Return `remainder === 1`

**extractIBAN(text)** — returns string or null (EXTR-02):
- Call `normalizeText(text)` first
- Regex pattern: `/\b([A-Z]{2}\d{2}(?:\s?[A-Z0-9]{4}){3,7}\s?[A-Z0-9]{0,3})\b/g` (case-insensitive via `.toUpperCase()` after match)
- For each candidate: strip spaces, uppercase, check length 15-34, run `validateIBAN`
- Return first valid candidate or null (per user constraint: first IBAN wins)

**parseAmountString(s)** — returns number or null (internal helper):
- Locale detection by last separator position: `lastComma > lastDot` means Dutch (comma = decimal), else English (dot = decimal)
- Dutch: remove dots (thousands), replace comma with dot
- English: remove commas (thousands)
- `parseFloat` the result, reject if `NaN` or `< 0.01` or `> 999999999.99`
- Round to 2 decimal places: `Math.round(value * 100) / 100`

**extractAmount(text)** — returns number or null (EXTR-03):
- Call `normalizeText(text)` first
- Regex pattern anchored to currency signals: `/(?:€|EUR|euro)\s*([\d.,]+)|(?:te\s+betalen|totaal|total|bedrag|amount\s+due)\s*:?\s*([\d.,]+)/gi`
- Iterate all matches with `pattern.exec(text)`, parse each with `parseAmountString`
- Return the LAST valid match (invoice totals appear at bottom)

**extractName(text, ibanIndex)** — returns string or null (EXTR-04):
- Call `normalizeText(text)` first
- Step 1 — Label matching: search for patterns like `/(?:t\.?\s?n\.?\s?v\.?|begunstigde|ten\s+name\s+van|naam|name|beneficiary)\s*:?\s*(.+)/gi`. Return first capture group trimmed.
- Step 2 — IBAN proximity fallback: if `ibanIndex` is provided, look at the non-empty trimmed line immediately before the IBAN position in the text. If it does not look like a label (no colon-only lines, no "rekeningnummer", no "IBAN" keyword), return it as candidate name.
- Step 3 — If nothing found, return null (user edits in Phase 2)
- Enforce 70-char EPC limit: `name.substring(0, 70)` on any result before returning (FLDS-03)

All functions must be accessible from the browser console (attached to `window` or declared at script top-level scope).

Do NOT use `import`/`export` — use plain function declarations at top-level scope.
Do NOT add any `fetch`, `XMLHttpRequest`, or network calls (GENL-03).

**Test structure:**
```javascript
function runTests() {
  let passed = 0, failed = 0;
  function assert(label, actual, expected) {
    if (actual === expected) {
      console.log('PASS: ' + label);
      passed++;
    } else {
      console.error('FAIL: ' + label + ' — expected ' + JSON.stringify(expected) + ', got ' + JSON.stringify(actual));
      failed++;
    }
  }
  // ... test cases ...
  console.log('\n' + passed + ' passed, ' + failed + ' failed');
  return { passed, failed };
}
```
