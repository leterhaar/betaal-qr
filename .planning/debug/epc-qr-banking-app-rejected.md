## Evidence

- timestamp: 2026-03-23
  checked: formatEPCPayload implementation in index.html lines 172-191
  found: lines array has 12 elements ['BCD','002','1','SCT','','name','iban','EUR25.50','','','',''] joined with '\n'. Last element is '' (empty string), so the joined result ends with '\n' (the separator between element 11 and the empty element 12). Final payload has a trailing newline.
  implication: EPC069-12 v3.1 spec explicitly states "The last populated element is not followed by any character or element separator." A trailing newline after the final empty field violates this. Some banking app parsers reject payloads with trailing newlines.

- timestamp: 2026-03-23
  checked: Dutch banking app EPC QR support via web research
  found: Rabobank and ABN AMRO do NOT support EPC QR codes at all. Only ING, Bunq, Knab, ASN, and SNS Bank support EPC QR. Rabobank/ABN AMRO represent ~60% of Dutch market and use iDEAL/Wero QR standard instead.
  implication: If user tested with Rabobank or ABN AMRO app, the QR will NEVER work regardless of payload correctness. This is the "Dutch standard" the user mentioned — those banks use a different standard entirely.

## Resolution

root_cause: Two compounding issues: (1) formatEPCPayload used lines.join('\n') with a trailing empty element (Line 12 = ''), producing a trailing \n that violates EPC069-12 v3.1 which requires "the last populated element is not followed by any character or element separator." (2) Rabobank and ABN AMRO do not support EPC QR standard at all — they use iDEAL/Wero QR instead — so testing with those apps will always fail regardless of payload correctness.

fix: Added while loop in formatEPCPayload to strip trailing empty elements from the lines array before join('\n'), ensuring the payload ends at the last populated field with no trailing newline. Updated tests to reflect correct behavior: 8 lines when amount is set but no remittance, 7 lines when amount is null, new assertion that payload does not end with '\n'. Also added test confirming remittance produces 11 lines.

verification: Logic verified by tracing through all test cases. Requires human confirmation: (1) run runTests() in browser to confirm all tests pass, (2) scan resulting QR with ING/Bunq/Knab/ASN/SNS app (NOT Rabobank or ABN AMRO which do not support EPC QR).
